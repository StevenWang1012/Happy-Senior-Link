「樂齡智享 (Happy Senior Link)」產品規劃書 V1.0

一、 產品核心定位

銀髮族專屬的「真實生活經驗種草社群」。透過去中心化的 Web 平台與 LINE Bot 雙端聯動，聚焦於本地生活、大陸漫遊、健康養生的真實分享。

1. 核心價值

無門檻：大字體、極簡操作、語音輸入，專為 55-75 歲長輩設計。

真經驗：去政治化，只聊實用生活資訊，由熟人與專家背書。

零成本：利用 Supabase、Cloudinary、GitHub Pages 等免費層級資源實現冷啟動。

二、 目標用戶畫像

1. 核心用戶 (55-75 歲台灣銀髮族)

特徵：會用 LINE、重視性價比與安全性、有大陸探親或旅遊需求。

痛點：數位產品字太小、操作太複雜、怕詐騙、缺乏同齡人的大陸攻略。

四、 視覺與交互規範 (UX Standards)

配色：主色 #E64134 (高對比紅)，背景 #F9F9F9 (暖灰)。

字體：內文預設 22px，標題 26px 以上。

佈局：單欄瀑布流 (3:4 圖片比例)，仿小紅書質感但保留大操作區域。

回饋：所有按鈕具備 :active 縮放效果，增加操作實感。

五、 核心功能板塊

1. 本地生活 (Local Life)

銀髮友善餐廳、無障礙公園步道、超商隱藏優惠。

特色：一鍵導航、一鍵列印。

2. 大陸漫遊 (China Roam)

支付寶/微信支付實戰教學、大陸無障礙交通指南、探親實用技巧。

特色：真實體驗分享，排除官方宣傳色調。

3. 健康不求人 (Health)

居家復健動作、養生食譜。

規範：強制顯示「非醫療建議」免責聲明。

六、 營運與安全策略

1. 內容審核機制

模式：LINE Bot 寫入 -> 人工後台審核 -> Web 前端顯示。

過濾：嚴格禁止政治、醫療誇大、詐騙連結。

2. 資安防護

Supabase RLS：預設僅開啟 SELECT (讀取) 權限給匿名用戶。

數據隱私：不主動收集長輩敏感個資。

七、 執行優先順序

第一階段 (MVP)：完成 Web 雙端連線，實現 Supabase 內容顯示 (已完成)。

第二階段 (發布流)：串接 LINE Bot -> Supabase 的自動化發布流程。

第三階段 (大陸專題)：針對大陸漫遊開發專屬篩選頁面與支付教學懶人包。

第四階段 (冷啟動)：與社區大學、里長辦公室合作獲取首批種子用戶。

八、 財務分析

初期營運費用：$0 (全數使用免費層級)。

預計升級節點：當月活躍用戶 (MAU) 超過 1,000 人或圖片儲存量超過 25GB。
